    public class SynchronizedTest {
    public volatile static boolean flag = true;

    public static void main(String[] args) {

    C c = new C();
new Thread(c::a).start();

    // new Thread(c::b).start(); //方法引用无法抛出异常
    }
    }

class C {
public synchronized void a() {

    System.out.println("进入a");
    while (SynchronizedTest.flag) {
        try {
            System.out.println("a放弃锁");
            wait(0); //wait(0)也相当于wait()，wait(timeout) 如果timeout过去后，亦然没有被notify.notifyAll唤醒或者被其他线程interrupt则，会自动往下运行。
            System.out.println("a被唤醒");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }

    System.out.println("flag为false 退出循环");

}


public synchronized void b() {

    System.out.println("进入b");
    try {
        Thread.sleep(5000);
    } catch (InterruptedException e) {
        e.printStackTrace();
    } finally {
    }

    SynchronizedTest.flag = false;

    System.out.println("b唤醒并退出");
    notifyAll();
}

}

yield

yield中文翻译为让步。也就是说调用了该方法的当前线程会暂时让出时间片，让其他线程有机会执行。该方法是Thread类中的静态本地方法，
需要注意的是，调用了该方法并不一定意味着当前线程一定会让步，yield让该线程从运行状态转换到可运行态，有可能操作系统继续调度了
该线程执行，那么有可能这个方法看起来就没起作用。

了解Java垃圾自动回收

发表于 2018-03-17 | 分类于 java

在虚拟机中，释放那些不再被使用的对象所占用空间的的过程称作垃圾回收，简称GC。

垃圾回收的三个难题：

  • 如何确定哪些对象不再被使用？
  • 什么时候进行回收？
  • 如何回收？

如何确定对象无用

  1. 引用计数算法

引用计数算法简单的说就是在给对象添加一个引用计数器，每次被引用则该计数器加1，当引用失效时，计数器减1，计数器值为0的表示该对象不再被使用。
该算法被Python等许多脚本语言采用来管理内存，但java中并没有使用该算法。因为该算法存在许多弊端，比如很难解决对象之间循环引用的问题。

java notify和notifyAll区别

发表于 2018-03-15 | 分类于 java

从字面上意思来说，notify是唤醒一个处于该对象wait的线程
而notifyAll是唤醒所有处于该对象wait的线程。
但是唤醒不等于就能执行了，需要得到锁对象才能有权利继续执行，而锁只有一把，所以多个线程被唤醒时需要争取该锁。
那你也许会想，既然只有一把锁，那我每次用notify唤醒一个不就行了，这样都省的争了，notifyAll还有存在的意义吗？

线程和进程

发表于 2018-03-14 | 分类于 操作系统

内核级线程:
就是直接由操作系统内核支持，这种线程由内核来完成线程切换，以及由内核通过调度器来调度，并
将线程的任务映射到各个处理器上。内核每个线程在核心空间设置一个线程控制块，设置了内核线程的系统，其调度是以线程为单位进行的。
用户一般不会直接使用内核级线程，而是去使用内核线程的一种高级接口—-轻量级进程，就是我们通常意义所讲的线程，它需要内核线程支持，所以
通常内核级线程和轻量级进程都是一对一模型。

优点:在多处理器或者多核处理器系统中，内核能通过调度同一进程中多个线程并行执行到多个处理器核心中，如果进程的一个线程阻塞，不会影响该进程其他线程，系统内核会调度
另一个线程运行。
缺点:正因为把工作交给了操作系统内核，所以几乎所有工作包括线程的创建，切换等都需要进行系统调用，在用户态和内核态切换会影响效率。

文件存储方式

  • 顺序存取：进程在这些系统中可以从头顺序读取文件的全部，但是不能跳过某些内容。在存储介质为磁带时，顺序存取是很方便的。
    顺序存取类似于链表节点的访问，想要获得第n个节点，必须从头节点开始访问前N-1个节点的指针域。

  • 随机存取:以任何次序读取其中字节的记录，类似于数组，可以直接索引访问指定单元。常见的随机存取读取文件方式如read指定开始读文件位置和
    seek方法设置当前位置。
