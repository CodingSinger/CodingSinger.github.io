了解Java垃圾自动回收

发表于 2018-03-17 | 分类于 java

在虚拟机中，释放那些不再被使用的对象所占用空间的的过程称作垃圾回收，简称GC。

垃圾回收的三个难题：

  • 如何确定哪些对象不再被使用？
  • 什么时候进行回收？
  • 如何回收？

如何确定对象无用

  1. 引用计数算法

引用计数算法简单的说就是在给对象添加一个引用计数器，每次被引用则该计数器加1，当引用失效时，计数器减1，计数器值为0的表示该对象不再被使用。
该算法被Python等许多脚本语言采用来管理内存，但java中并没有使用该算法。因为该算法存在许多弊端，比如很难解决对象之间循环引用的问题。

线程和进程

发表于 2018-03-14 | 分类于 操作系统

内核级线程:
就是直接由操作系统内核支持，这种线程由内核来完成线程切换，以及由内核通过调度器来调度，并
将线程的任务映射到各个处理器上。内核每个线程在核心空间设置一个线程控制块，设置了内核线程的系统，其调度是以线程为单位进行的。
用户一般不会直接使用内核级线程，而是去使用内核线程的一种高级接口—-轻量级进程，就是我们通常意义所讲的线程，它需要内核线程支持，所以
通常内核级线程和轻量级进程都是一对一模型。

优点:在多处理器或者多核处理器系统中，内核能通过调度同一进程中多个线程并行执行到多个处理器核心中，如果进程的一个线程阻塞，不会影响该进程其他线程，系统内核会调度
另一个线程运行。
缺点:正因为把工作交给了操作系统内核，所以几乎所有工作包括线程的创建，切换等都需要进行系统调用，在用户态和内核态切换会影响效率。

文件存储方式

  • 顺序存取：进程在这些系统中可以从头顺序读取文件的全部，但是不能跳过某些内容。在存储介质为磁带时，顺序存取是很方便的。
    顺序存取类似于链表节点的访问，想要获得第n个节点，必须从头节点开始访问前N-1个节点的指针域。

  • 随机存取:以任何次序读取其中字节的记录，类似于数组，可以直接索引访问指定单元。常见的随机存取读取文件方式如read指定开始读文件位置和
    seek方法设置当前位置。
