以例子中的int age参数解析来说，对应的解析器解析行为如下：

  1. NamedValueInfo对象保存了该参数的名称以及是否必需、默认值,可以在@RequestParam等注解中设置。
  2. 然后根据该参数信息去请求中解析得到本次传递的值。
  3. 如果请求中没有传递该参数的值，则先判断NamedValueInfo中该参数是否有设置过默认值，如果有，则设置为默认值。
  4. 如果没有默认值并且要求传递该参数(NamedValueInfo.requiredtrue)则handleMissingValue方法会扔出一个ServletRequestBindingException异常。
  5. 否则的话就运行handleNullValue方法，该方法为未找到的参数赋值为空，但是如果是基本类型的话就会扔出IllegalStateException异常。
  6. 如果arg不为空，则说明请求包含该参数，则获得对应的参数绑定器并且如果需要类型转化则进行类型转化，最后将转化后的参数返回。

总结

为了解析请求的参数到对应方法参数，步骤如下：

  1. SpringMVC初始化时，RequestMappingHandlerAdapter类会把一些默认的参数解析器添加到argumentResolvers中。当SpringMVC接收到请求后首先根据url查找对应的HandlerMethod。
  2. 生成反射调用方法需要的Object数组参数，遍历HandlerMethod的MethodParameter数组(MethodParameter就是方法参数的一些信息的包装)
  3. 根据MethodParameter的类型来查找确认使用哪个HandlerMethodArgumentResolver
  4. 利用支持的HandlerMethodArgumentResolver将请求中所带的参数解析成当前遍历的MethodParameter对应类型的参数。
    其中包括类型的转化，对Bean类型的参数则需要特殊的处理，比如注入和包装。
  5. 将解析完的方法参数放入Object数组。

这里额外提一下，上面的参数绑定都通过了参数名称绑定。那SpringMVC是如何获得参数名称的呢?

this.parameterNameDiscoverers主要包含LocalVariableTableParameterNameDiscoverer和StandardReflectionParameterNameDiscovere

  1. 通过Parameter.getName()，但是编译时需要指定参数javac -parameters，不然得到的名称仍然是arg0,arg1…
  2. LocalVariableTableParameterNameDiscoverer 类中是用 ASM 字节码操纵框架读取字节码文件获取真实在 java 代码中定义的真实参数名
