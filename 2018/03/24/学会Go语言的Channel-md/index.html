学会Go语言的Channel

发表于 2018-03-24 | 分类于 Golang

channel的声明和初始化

Channel类型的定义格式如下：
ChannelType = ( “chan” | “chan” “<-“ | “<-“ “chan” ) ElementType .
它包括三种类型的定义。可选的<-代表channel的方向。如果没有指定方向，那么Channel就是双向的，既可以接收数据，也可以发送数据。

认识make(Type IntegerType,size IntegerType)

没有缓冲的channel （size 为0）可以用来处理某些同步问题，因为sender阻塞直到该channel有人接收，同理，recevier也阻塞直到sender发送

如果size不为0，则当channel满时，sender会阻塞直到有一个接受者从channel中取走数据。总的来说,channel就是满时发送者阻塞，空时接受者阻塞。

close channel

用close关闭channel之后，将无法再进行发送数据到此channel否则将会报错，但是能继续接受数据，就像官方对该API的介绍用close关闭channel之后，将无法再进行发送数据到此channel否则将会报错，但是能继续接受数据，就像官方对该API的介绍

// The close built-in function closes a channel, which must be either
// bidirectional or send-only. It should be executed only by the sender,
// never the receiver, and has the effect of shutting down the channel after
// the last sent value is received. After the last value has been received
// from a closed channel c, any receive from c will succeed without
// blocking, returning the zero value for the channel element. The form
// x, ok := <-c
// will also set ok to false for a closed channel.
主要意思就是close方法应该被sender调用，并且channel关闭之后不能再发送数据到该channel,关闭之后能继续接受数据从该channel中，并且即使该channel没有数据，也会无阻塞地从该channel中取出对应类型的零值。

用range遍历channel

range遍历阻塞直到channel中有数据。

在结束放入的同时，立刻发生了panic:fatal error: all goroutines are asleep - deadlock!

因为range遍历channel会一直尝试接受数据直到该channel关闭。因此此处for range将无限制的阻塞下去，造成死锁

close channel 等于 channel = nil 吗

这里我们记住两个性质：

  • 已经关闭了的channel在无数据可取后会无阻塞地返回对应类型的零值，并且用val,ok := <-c来接受,ok会被置为false

  • 对一个赋值为空的channel引用进行操作将会阻塞。这个性质在下面将有用处。
